
# **CamStem: Comprehensive Project Documentation**

### **Project Name**  
**CamStem**

### **Project Purpose**  
CamStem is a desktop application for audio processing, built with Electron, Tailwind CSS, and JavaScript. It allows users to upload an audio file, enter a file path, or paste a link from platforms like Spotify, SoundCloud, or YouTube. The app then downloads or imports the audio and uses a stem-splitting library (such as Demucs) to separate the audio into individual stems (e.g., vocals, instruments). The output, along with the original file, is saved in a folder specified by the user. CamStem is ideal for users who want to isolate different components of a song for remixing, analysis, or production.

## **Detailed File Summaries**

### **`src/assets/`**
- **`logo.ico`**: The app icon used in the Electron window and taskbar. Electron prefers `.ico` format for compatibility with Windows; any attempt to use `.png` may result in the icon not appearing.

### **`src/configurations/`**
- **`tailwind.config.js`**: Tailwind configuration file where you define:
  - **`content`** paths: Specifies which files Tailwind should scan for CSS classes. We use `"./src/**/*.{html,js}"` to cover all HTML and JS files within `src`.
  - **Custom Styles**: Defines custom colors, font sizes, and box shadows for a modern, "bubbly" UI. For example:
    - `error-red`: `#ef4444`
    - `blue-primary`: `#3b82f6`
    - `blue-hover`: `#2563eb`
    - Custom font sizes like `smaller` and `large` for flexible text styling.

### **`src/cosmetic_process/`**
- **`input.css`**: The source CSS file that Tailwind processes to generate `tailwind.css`. This file includes Tailwind directives:
  ```css
  @tailwind base;
  @tailwind components;
  @tailwind utilities;
  ```
- **`tailwind.css`**: The compiled CSS file generated from `input.css`. It includes all necessary styles based on Tailwind's scan of the files listed in `tailwind.config.js`. It is linked in the HTML files (`login.html` and `dashboard.html`) to style the UI.

### **`src/login_process/`**
- **`auth.js`**: Handles authentication logic. Contains placeholder data for user accounts, including username and password validation, and defines specific error messages:
  - **`USER_NOT_FOUND`**: When the username does not match any existing account.
  - **`INCORRECT_PASSWORD`**: When the password is incorrect.
  - **`ACCOUNT_INVALID`**: For invalid or locked accounts.
  - **`verifyLogin`**: Function that checks for correct username and password and returns the appropriate error if authentication fails.
- **`login.html`**: The login page where users input their credentials. Includes:
  - **Input fields** for `username` and `password`.
  - A **"Forgot Password"** button.
  - Styled using `tailwind.css`, with elements designed to match the "bubbly" look of the app. Connected to `renderer.js` for login functionality.

### **`src/main_process/`**
- **`dashboard.html`**: The main interface displayed after login. Includes:
  - **Buttons**: Options for “Download Audio Only,” “Split Stems,” and “About CamStem.”
  - **Logout button**: Triggers `logout()` in `renderer.js`, sending the user back to `login.html`.
  - Linked to `tailwind.css` for consistent styling with the login page.

### **`src/root_process/`**
- **`main.js`**: The main entry point for the Electron app. Responsibilities include:
  - **Setting up the BrowserWindow**:
    - Title: **"CamStem v0.1"**
    - Icon: **`logo.ico`**
    - **Window size**: Starts at 1024x768 for a consistent experience.
  - **App Menu**:
    - Standard menus for `File`, `Edit`, `View`, and `Window`.
    - Custom **Help menu** with links to:
      - **GitHub Documentation**: `"https://github.com/davidcamick/CamStem"`
      - **Instagram Support**: `"https://www.instagram.com/davidcamick/"`
    - **WebPreferences**: `preload` script is specified, `contextIsolation` is enabled for security, and `nodeIntegration` is turned on to allow Node modules like `path` in `preload.js`.
  - **Window Management**:
    - Handles app behavior for closing, minimizing, and reactivating windows.
- **`preload.js`**: Manages secure communication between frontend and backend using Electron’s `contextBridge`:
  - Loads `auth.js` from `login_process` and exposes the `verifyLogin` function as `authAPI.verifyLogin` to the renderer process.
  - Includes error handling to log any failures in loading `auth.js`.
- **`renderer.js`**: Handles front-end interactions, particularly:
  - **Login Verification**: Uses `window.authAPI.verifyLogin()` to check credentials. If login is successful, the user is redirected to `dashboard.html`; otherwise, an error is displayed.
  - **Logout Handling**: Sends the user back to `login.html` upon logging out.

## **How the Program Works**

1. **App Launch**:
   - Electron starts with `main.js`, creating a BrowserWindow with a specified title and icon, loading `login.html` as the starting page.
   - The `preload.js` script is attached to securely expose backend functionality.

2. **Login Flow**:
   - Users enter credentials on the login page (`login.html`), which `renderer.js` verifies by calling `window.authAPI.verifyLogin()`.
   - If credentials are correct, the user is redirected to `dashboard.html`. If incorrect, the app displays an error message near the "Forgot Password" button.

3. **Dashboard Functionality**:
   - The dashboard page (`dashboard.html`) provides users with options to download or split audio. Each button initiates a relevant function (e.g., launching the download process or calling a stem-splitting function).
   - The logout button takes the user back to `login.html`.

4. **Help Menu**:
   - A custom Help menu with links to GitHub documentation and Instagram for support, allowing easy access to resources.

5. **Error Handling**:
   - Authentication errors are managed in `auth.js` and are displayed on `login.html` via `renderer.js`.

## **Common Preferences and Requests**

### **File Structure and Organization**
- The project is organized into clearly defined folders for different functionalities (`assets`, `configurations`, `cosmetic_process`, etc.).
- **Modular Structure**: Each folder focuses on specific tasks, such as configuration, authentication, main functionality, and visual styling. This makes it easy to manage and update different components without affecting unrelated parts of the project.

### **Styling Preferences**
- **Bubbly, Modern Aesthetic**: Rounded buttons, a clean layout, and responsive design are emphasized.
- **Tailwind Customization**: Specific colors (e.g., `blue-primary` for primary buttons, `error-red` for error messages) and font sizes (`smaller` and `large`) are defined for a cohesive look.
- **Tailwind CSS Workflow**: `input.css`

 is compiled into `tailwind.css` via the `npm run build:css` command, which uses the configuration in `src/configurations/tailwind.config.js`.

### **Electron Preferences**
- **Icon Requirements**: Only `.ico` format is supported for the app icon in Windows.
- **Help Menu**: The Help menu includes GitHub documentation and Instagram support links.
- **Cache Clearing**: Occasionally, the cache needs to be cleared in Electron to avoid stale styles.

### **Git Workflow**
- **Concise Commit Messages**: Example commit:
  ```bash
  git add .
  git commit -m "Updated login flow and added Help menu"
  git push
  ```

### **Troubleshooting and Testing Patterns**
- **Path Verification**: Paths for all files, especially CSS and configuration files, are crucial for the app to work smoothly.
- **Rebuilding Tailwind CSS**: Run `npm run build:css` after any changes to Tailwind configuration or `input.css`.
- **Electron Cache Clearing**: Add `win.webContents.session.clearCache()` in `main.js` to reset cached assets, if styling issues persist.
