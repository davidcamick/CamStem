
## Project Overview

### Project Name
**CamStem**

### Project Purpose
CamStem is a desktop application built using Electron, Tailwind CSS, and JavaScript. Users enter a file path, Spotify/SoundCloud/YouTube link, or upload audio files (e.g., MP3, WAV). The application detects the input, processes it (e.g., downloads, imports), and uses a stem-splitting library (e.g., Demucs) to separate audio into stems, outputting them into a chosen folder along with the original file.

### Key Features Implemented
- **Login Functionality**: Users must enter a valid username and password to access the main dashboard.
- **Dashboard**: Once logged in, users access a dashboard with options like "Download Audio Only" and "Split Stems."
- **Secure Frontend-Backend Interaction**: Uses Electron‚Äôs `contextBridge` and `preload.js` to expose backend functions securely.
- **Styling**: Tailwind CSS for a modern, responsive, and "bubbly" design with customized styles for buttons, error messages, and popups.

## File Summaries and Functionality

### `root_process/`
Contains files managing the main Electron process and the interaction between frontend and backend securely.

- **`main.js`**: Initializes the Electron application.
  - Sets up the main window, loads `login.html` from `login_process`, and links `preload.js`.
  - Path to `preload.js` is updated to match the new structure.
- **`preload.js`**: Uses Electron's `contextBridge` to expose secure backend functionality.
  - Loads and exposes `verifyLogin` from `auth.js` using contextBridge.
  - Contains a `try-catch` block to handle errors in loading `auth.js`.
- **`renderer.js`**: Handles all UI interactions, login logic, and page navigation.
  - Uses `window.authAPI.verifyLogin()` (from `preload.js`) to check login credentials.
  - Manages login redirection to `dashboard.html` and logout redirection to `login.html`.

### `login_process/`
Files related to login and authentication.

- **`login.html`**: HTML for the login page.
  - Includes fields for `username` and `password`.
  - Contains a "Forgot Password" button that is always visible, smaller, and styled to match the app.
  - Links to `tailwind.css` and `renderer.js`.
- **`auth.js`**: Provides authentication logic.
  - Contains placeholder data for `user` accounts, including username/password and account validity.
  - `verifyLogin` function checks for username and password correctness and account validity.
  - Errors include `"USER_NOT_FOUND"`, `"INCORRECT_PASSWORD"`, and `"ACCOUNT_INVALID"`.

### `cosmetic_process/`
Contains all styling files.

- **`tailwind.css`**: The compiled CSS file generated by Tailwind.
  - Should be updated by running `npx tailwindcss -i ./src/cosmetic_process/input.css -o ./src/cosmetic_process/tailwind.css`.
- **`input.css`**: The source CSS file to build `tailwind.css`.
  - Includes `@tailwind base;`, `@tailwind components;`, and `@tailwind utilities;`.

### `configurations/`
Holds configuration files for the project.

- **`tailwind.config.js`**: Configures Tailwind to scan specific files for class usage.
  - `content` paths include `login.html`, `dashboard.html`, and JavaScript files in `root_process` and `login_process`.
  - Customizations for colors, font sizes, and shadows to give a ‚Äúbubbly‚Äù and modern look.

### `main_process/`
Main functionality files, including the user dashboard.

- **`dashboard.html`**: The main dashboard page after successful login.
  - Contains three buttons: "Download Audio Only," "Split Stems," and "About CamStem."
  - Includes a Logout button, linked to `logout()` in `renderer.js`.
  - Links to `tailwind.css` and `renderer.js`.

## Summary of How the Program Works

1. **Launch Process**:
   - `main.js` initializes Electron, loading `login.html` through `BrowserWindow`.
   - Uses `preload.js` to securely expose `verifyLogin` from `auth.js`.
   
2. **Login Process**:
   - Users enter credentials on `login.html`.
   - `renderer.js` verifies credentials by calling `window.authAPI.verifyLogin()`.
   - If credentials are correct, the user is redirected to `dashboard.html`. If incorrect, an error message is shown.
   
3. **Dashboard and Features**:
   - On successful login, `dashboard.html` provides options to download or split audio and includes an "About" section.
   - A "Logout" button redirects the user back to `login.html`.

4. **Error Handling**:
   - `auth.js` defines error types for login (e.g., incorrect password, invalid account).
   - `renderer.js` displays the appropriate error message below the "Forgot Password" button.
